\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

Modern computer architectures execute concurrent programs on shared memory, where subtle hardware and compiler behaviors can lead to bugs and security vulnerabilities. To reason about such behaviors systematically, the Cat specification language represents program executions as graphs of binary relations, enabling concise axiomatic definitions of weak memory models. Many memory models---including those for RISC-V, Arm, x86, and the Linux Kernel Memory Model (LKMM)---are already defined in Cat. However, mechanizing these models in a proof assistant and constructing formal proofs about them remain largely orthogonal problems, with few existing solutions addressing both simultaneously.

In this thesis, we present LeanCat, a domain-specific proof framework for mechanizing weak memory models in Lean 4. LeanCat parses existing Cat specifications directly and imports their semantics into Lean 4 via its metaprogramming framework, eliminating the need to translate or rewrite memory model definitions. The framework provides the necessary primitives, abstractions, and library theorems to help users prove properties such as whether one memory model is strictly weaker than another.

We implement a core subset of the Cat language targeting LKMM and demonstrate LeanCat through two case studies: checking litmus test executions and formally comparing memory models. In particular, we present a machine-checked proof that Sequential Consistency (SC) is strictly stronger than Total Store Order (TSO), reducing the comparison to a monotonicity argument over acyclicity constraints. Our results show that LeanCat significantly reduces the mechanization effort and boilerplate required for weak memory model proofs, and that the approach generalizes naturally to other model comparisons.
